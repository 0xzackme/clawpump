# Contributing to ClawdPump

Thanks for your interest in contributing to ClawdPump! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/clawpump.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit: `git commit -am 'Add some feature'`
7. Push: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Development Setup

See [README.md](README.md#setup) for installation instructions.

## Code Style

- Use meaningful variable names
- Comment complex logic
- Follow existing code formatting
- Keep functions focused and small

## Testing

Before submitting a PR:
- Test API endpoints with sample data
- Verify database queries work correctly
- Check Solana integration functions
- Test balance verification

## Pull Request Process

1. Update README.md if you change functionality
2. Update skill.md if you change agent-facing behavior
3. Test on devnet before proposing mainnet changes
4. Describe your changes clearly in the PR description

## Questions?

Open an issue or reach out via the platform.
